# 自動更新の設定手順（GitHub Actions + Pages）

これを設定すると、**PCを起動しなくても毎朝7時頃（平日）にクラウドが自動でレポートを更新**し、
スマホからいつでも最新版が見られるようになります。作業は初回の30分だけです。

## 仕組み

```
GitHub Actions（毎朝7:00 JST、クラウド上で自動実行）
  → run.py 実行 → index.html 生成 → リポジトリにコミット
  → GitHub Pages が公開: https://<ユーザー名>.github.io/<リポジトリ名>/
  → スマホの「ホーム画面に追加」でアプリ化（PWA）
```

## 手順

### 1. GitHubアカウント作成（未取得の場合）

https://github.com/signup で無料アカウントを作成。

### 2. リポジトリ作成

1. https://github.com/new を開く
2. Repository name: 推測されにくい名前にする（例: `asa-repo-x7k2`）※URLの一部になる
3. **Public** を選択（無料プランではPagesはPublicのみ）
4. 「Create repository」

### 3. コードをアップロード（push）

PCに [Git](https://git-scm.com/downloads) を入れて、kabu-app フォルダーで:

```bash
cd kabu-app
git init
git add .
git commit -m "initial"
git branch -M main
git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
git push -u origin main
```

※ `holdings.csv`（保有銘柄）と `data/`（キャッシュ）は .gitignore 済みで公開されません。

### 4. Actionsの書き込み権限を有効化

リポジトリの **Settings → Actions → General → Workflow permissions** で
**「Read and write permissions」** を選択して Save。
（ワークフローがレポートをコミットするために必要）

### 5. GitHub Pages を有効化

**Settings → Pages → Build and deployment**
- Source: **Deploy from a branch**
- Branch: **main** / **(root)** → Save

数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` が公開されます。

### 6. 動作テスト（手動実行）

リポジトリの **Actions タブ → daily-report → Run workflow** ボタンで手動実行。
緑のチェックが付いたら、上記URLをブラウザで開いてレポートが出ることを確認。

### 7. スマホでアイコン化

スマホのブラウザで公開URLを開き:
- **iPhone (Safari)**: 共有ボタン → 「ホーム画面に追加」
- **Android (Chrome)**: メニュー(⋮) → 「ホーム画面に追加」

以降、ホーム画面のアイコンからアプリのように起動できます。

## 運用メモ

- 実行は平日の朝7:00 JST設定。GitHubのcronは**数分〜15分程度遅れる**ことがあります
- yfinanceがGitHubのIPをブロックして取得に失敗する日があり得ます。その場合もキャッシュ
  （Actionsのcacheに保持）で前日データのレポートが出ます。失敗銘柄はレポートに表示されます
- 保有銘柄チェックは `holdings.csv` がローカル専用のため、**公開ページには出ません**
  （PC上で `python run.py` を実行した時のみ表示）。これは秘匿方針どおりの仕様です
- 祝日（東証休場日）は前日と同じ内容になります（祝日カレンダー対応はPhase 2の残課題）
- config.yaml や universe.csv を変えたら `git add . && git commit -m "update" && git push` で反映

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| Actionsが赤くなる | Actionsタブ→失敗した実行→ログを確認。権限エラーなら手順4を再確認 |
| ページが404 | 手順5のPages設定を確認。反映に数分かかることも |
| レポートが更新されない | Actionsタブで朝の実行が動いたか確認。60日間リポジトリに変更がないとcronが自動停止するため、Actionsタブの通知から再有効化 |
